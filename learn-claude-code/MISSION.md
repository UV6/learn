# Mission: Master Agent Harness Engineering

## Why
I'm building a coding agent in TypeScript and need to understand how production-grade agent harnesses work from the inside out. I want the architectural patterns — the agent loop, tool dispatch, permission systems, context management, team coordination — so I can design my own framework correctly, not just copy existing ones. Beyond coding agents, I want the general harness principles that apply to any domain (agriculture, healthcare, education, manufacturing).

## Success looks like
- I can explain every harness mechanism (s01-s20) in my own words, including why it exists and when to use it
- I have a working TypeScript implementation of the core harness patterns, not just the Python demos
- My coding agent's architecture is grounded in the harness philosophy: the model provides intelligence, the harness provides the world
- I can look at any agent product and identify which harness mechanisms it has (and which it's missing)

## Constraints
- 1-2 hours per day, aim to complete all 20 sections in 1-2 weeks
- The course code is in Python; I'll translate key patterns to TypeScript as I go
- Learning is self-directed — I need lessons that are self-contained and actionable

## Out of scope
- Training/fine-tuning models (this is about harness, not model training)
- Frontend UI for agent products
- Production deployment, scaling, or observability (for now)
