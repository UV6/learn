0001 — Agent Loop：一个循环就够了

课程：s01_agent_loop  |  源码：code.py  |  参考：术语表

心智转换：你不是在"开发 Agent"

在写第一行代码之前，先把一件事焊死在脑子里：

Agency（感知、推理、行动的能力）来自模型训练，不是来自外部代码的编排。 模型是驾驶者，Harness 是载具。你写的是载具。

这意味着什么？你不需要在代码里写决策树告诉模型"什么时候该读文件"、"什么时候该跑测试"。 模型已经知道怎么推理了。你的工作只有两件事：

  1. 给模型手和眼睛（工具：bash、read、write、browser…）
  2. 给模型一个能持续行动的循环（调用工具 → 看到结果 → 继续推理 → 调用工具 → …）

历史已经证明了这个模式：DeepMind 的 DQN（2013）、OpenAI Five（2019）、AlphaStar（2019）、腾讯绝悟（2019）、Claude Code（2024）—— 每一个里程碑都是模型提供智能 + Harness 提供环境。模型在变，循环不变。

核心循环：不到 30 行

把 agent 的工作过程翻译成人话：

  1. 用户发来一个问题
  2. 把问题 + 可用工具列表发给模型
  3. 模型说"我要用工具 X"还是"我做完了"？
  4. 如果要用工具 → 执行 → 把结果喂回去 → 回到第 2 步
  5. 如果说做完了 → 退出循环，返回结果

翻译成代码：

def agent_loop(messages):
    while True:
        # 1. 把消息和工具定义一起发给 LLM
        response = client.messages.create(
            model=MODEL, system=SYSTEM,
            messages=messages, tools=TOOLS,
            max_tokens=8000,
        )
        messages.append({"role": "assistant", "content": response.content})

        # 2. 模型不调工具 → 结束
        if response.stop_reason != "tool_use":
            return

        # 3. 执行模型要求的工具，收集结果
        results = []
        for block in response.content:
            if block.type == "tool_use":
                output = TOOL_HANDLERS[block.name](**block.input)
                results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": output,
                })

        # 4. 把工具结果喂回去，继续循环
        messages.append({"role": "user", "content": results})
关键洞察：Claude Code 的 src/query.ts 有 1729 行。 它的核心就是上面这 30 行 while True。剩下的 1699 行全是保护机制：token 超限恢复、 上下文压缩、流式工具执行、hook 中断处理、费用控制……但循环本身不变。

整个循环只有两个信号驱动：

信号含义循环动作
stop_reason == "tool_use" 模型举手说"我要用工具" 执行工具 → 结果喂回去 → 继续
stop_reason != "tool_use" 模型说"我做完了" 退出循环

为什么 bash 一个工具就够了

s01 只给模型一个工具：bash。模型可以执行任意 shell 命令。 读文件？cat file.py。写文件？echo "..." > file.py。找文件？find . -name "*.py"。

一个工具 + 一个循环 = 一个能工作的 Agent。粗糙，但完整。后面的 19 个章节都在这个循环上叠加机制， 循环本身从不改变。

+----------+ +-------+ +---------+ | User | ---> | LLM | ---> | Tool | | prompt | | | | execute | +----------+ +---+---+ +----+----+ ^ | | tool_result | +---------------+ (loop continues)

跟你的 TypeScript Agent 有什么关系

你在用 TypeScript 开发 coding agent。这个循环翻译成 TS 几乎一个字不改：

async function agentLoop(messages: Message[]): Promise<void> {
  while (true) {
    const response = await client.messages.create({
      model: MODEL,
      system: SYSTEM,
      messages,
      tools: TOOLS,
      max_tokens: 8000,
    });

    messages.push({ role: "assistant", content: response.content });

    if (response.stop_reason !== "tool_use") return;

    const results: ToolResult[] = [];
    for (const block of response.content) {
      if (block.type === "tool_use") {
        const output = await toolHandlers[block.name](block.input);
        results.push({
          type: "tool_result",
          tool_use_id: block.id,
          content: output,
        });
      }
    }

    messages.push({ role: "user", content: results });
  }
}

这 20 行就是你整个 agent 的骨架。后面每一章加一个机制，改的是 TOOLS、 加的是 TOOL_HANDLERS 的条目、或是在工具执行前后插逻辑，但 while (true) 永远不变。

检验理解

1. 在 agent loop 中，谁决定"何时调用工具、调用哪个工具"？
Harness 代码中的规则引擎
模型本身，基于训练学到的推理能力
用户每次手动指定
2. stop_reason 不等于 "tool_use" 时，循环应该怎么做？
重试一次，可能是网络错误
退出循环，模型已经完成任务
强制让模型调用工具
3. Claude Code 的 query.ts（1729 行）和 s01 的 code.py（30 行）是什么关系？
完全不同的架构，没有可比性
核心循环相同，多出来的 1699 行是保护机制
query.ts 用了一个更复杂的循环算法
