Agent Harness 术语表

所有课程共用的术语定义。随课程进展逐步扩充。
Agency 核心概念
感知环境、推理目标、采取行动的能力。来自模型训练（数十亿次梯度更新在行动序列数据上的结果），不是来自外部代码编排。Agency 是学出来的，不是编出来的。
Harness 核心概念
为 agent 模型提供可操作环境的一切代码和基础设施。Harness = Tools + Knowledge + Observation + Action Interfaces + Permissions。模型做决策，Harness 执行。模型是驾驶者，Harness 是载具。
Agent Loop 核心概念
Agent 的核心运行循环：while stop_reason == "tool_use" → 执行工具 → 结果喂回模型 → 继续推理。这个循环是每个 AI agent 的骨架，无论领域如何变化，循环本身不变。见 Lesson 0001。
Tool 机制
模型在环境中可以采取的一个行动。在 coding agent 中：bash、read_file、write_file、edit_file、glob、browser。每个工具是一个原子化的、可组合的行动单元。见 s02。
Tool Dispatch 机制
将模型的工具调用请求路由到对应处理函数的机制。通过 TOOL_HANDLERS 字典/map 实现查表分发。加新工具只需要注册一个新的 handler，循环代码完全不变。见 s02。
stop_reason 机制
LLM API 响应中的一个字段，表示模型为什么停止生成。值为 "tool_use" 表示模型想调用工具（循环继续），其他值（如 "end_turn"）表示模型已完成任务（循环退出）。
messages[] 机制
Agent loop 中累积的消息数组。包含 user 消息、assistant 响应（含 tool_use 块）、tool_result 消息。这是模型的"上下文"——它看到的一切。
Permission Pipeline 机制
工具执行前的三道闸门：1) 拒绝列表（永远禁止的操作），2) 规则匹配（需要审批的操作），3) 用户审批（暂停等待确认）。三道都没命中 → 直接执行。见 s03。
Hook 机制
挂在工具执行前后的扩展插口（PreToolUse / PostToolUse），不改主循环也能添加横切逻辑。见 s04。
TodoWrite 机制
让模型在执行前先列出计划步骤的工具。先计划后执行，显著提高复杂任务的完成率。见 s05。
Subagent 机制
派生子 agent 处理子任务，每个子 agent 有自己干净的 messages[]，只把最终结果带回主 agent。防止上下文噪声泄露，是模块化 agent 系统的基础。见 s06。
Context Compact 机制
四层压缩策略（snip / micro / budget / auto），在上下文窗口将要溢出前腾出空间。便宜的压缩先跑，贵的后跑。见 s08。
Context Window 概念
LLM 一次能"看到"的最大 token 数量。超出窗口的内容会被截断。Harness 的核心挑战之一就是在有限窗口内管理无限增长的对话历史。
Prompt Water Pipe Worker 概念
（提示词水管工）讽刺用语。指那些试图通过堆叠 if-else 分支、节点图、硬编码路由逻辑来"构建 Agent"的人，把 LLM 当成美化了的文本补全节点。Harness 工程师的对立面。
