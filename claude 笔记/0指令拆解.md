## /compact

**三步执行逻辑**
1. 批量清冗余（自动丢弃高 Token 内容）
直接删除全量历史 tool 输出：read文件全文、bash终端日志、grep/glob扫描结果、报错堆栈、过往调试冗余打印，这类是会话最耗 token 的部分，优先清空；只保留文件修改记录、变更清单。
2. LLM 生成结构化会话摘要
模型通读整段聊天，提炼关键信息：
已完成：代码改动、需求落地、已修复 BUG、敲定方案
待进行：剩余开发计划、遗留问题、约束条件
关键文件列表、核心配置、决策要点
丢弃反复试错、闲聊、无效讨论，数万 Token 对话浓缩为3k~8k 精简摘要。
3. 替换会话上下文，重启会话
删掉全部原始聊天记录，把生成的摘要作为新会话唯一初始上下文，后续对话基于摘要继续，Token 直接下降 60%~80%，释放大量上下文窗口。

**进阶用法**
```
# 常规压缩 
/compact 
# 定向压缩（指定保留内容，最优用法） 
/compact 保留cost-model数据表结构与SQL变更，丢弃测试调试日志
```

## /loop
/loop 时间间隔 + 你要循环做的事
`/loop` = 让 Claude 自动 “每隔一段时间重复执行同一个任务”，不用你一直盯着、反复发指令
/loop 每2分钟检查部署状态 
/loop 每1分钟查看服务日志有没有报错 
/loop 每5分钟拉取最新构建结果
停止：/loop stop
