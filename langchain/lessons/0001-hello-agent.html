Lesson 0001: Hello Agent

2026-06-26 · 预计时间 30 分钟 · 难度：入门

本课目标：安装 LangChain 环境，理解 Agent 是什么，用 create_agent 构建第一个能调用工具的智能代理。

1. Agent 是什么？

面试时最常见的第一个问题：「什么是 Agent？它和普通的 LLM 调用有什么区别？」

简单说：

  • 普通 LLM 调用：用户提问 → 模型回答。模型只能「说话」，不能「做事」。
  • Agent：用户提问 → 模型思考 → 决定调用哪个工具 → 观察工具结果 → 继续思考 → 最终回答。模型既能「说话」也能「做事」。

Agent 的核心循环（面试必背）：

用户输入 → LLM 推理 → 需要工具？→ 是 → 调用工具 → 获取结果 → LLM 再次推理
                    → 否 → 生成最终回答
面试要点：Agent = LLM + Tools + Orchestration Loop。LangChain 的 create_agent() 帮你封装好了这个循环。

python --version  # 应该 ≥ 3.10

创建项目并安装依赖：

3. 第一个 Agent：天气预报助手

我们将创建一个 Agent，它能调用 get_weather 工具来回答天气查询。

创建 agent.py：

import os
from dotenv import load_dotenv
from langchain.agents import create_agent
from langchain.chat_models import init_chat_model

load_dotenv()

# Step 1: 定义一个工具（Tool）
# 工具就是普通的 Python 函数 + type hints + docstring
def get_weather(city: str) -> str:
    """Get the current weather for a given city.

    Args:
        city: The name of the city to get weather for.
    """
    # 实际项目中，这里会调用天气 API
    # 现在我们用模拟数据
    weather_data = {
        "beijing": "晴朗，25°C，湿度 40%",
        "shanghai": "多云，28°C，湿度 65%",
        "tokyo": "小雨，22°C，湿度 80%",
    }
    return weather_data.get(city.lower(), f"找不到 {city} 的天气数据")

# Step 2: 初始化模型
model = init_chat_model(
    "gpt-4o-mini",  # 便宜够用；也可以用 "claude-sonnet-4-6"
    temperature=0,
)

# Step 3: 创建 Agent
agent = create_agent(
    model=model,
    tools=[get_weather],  # 把手头的工具交给 Agent
    system_prompt="你是一个友好的天气助手。用中文回答用户的问题。",
)

# Step 4: 运行 Agent
result = agent.invoke(
    {"messages": [{"role": "user", "content": "北京今天天气怎么样？"}]}
)

# Step 5: 查看结果
print("=== Agent 完整消息历史 ===")
for msg in result["messages"]:
    print(f"[{msg.type}] {msg.content[:200]}...")

运行它：

python agent.py

你会看到类似这样的输出：

=== Agent 完整消息历史 ===
[human] 北京今天天气怎么样？...
[ai] ...
[tool] 晴朗，25°C，湿度 40%...
[ai] 北京今天天气晴朗，温度 25°C，湿度 40%，是个不错的天...

4. 发生了什么？

当你调用 agent.invoke() 时，LangChain 在背后执行了 Agent Loop：

  1. human 消息 — 用户输入「北京今天天气怎么样？」
  2. LLM 推理 — 模型看到有 get_weather 工具可用，判断需要调用它，返回 tool_call
  3. Tool 执行 — LangChain 自动调用 get_weather("beijing")，结果返回「晴朗，25°C」
  4. LLM 再次推理 — 模型拿到工具结果，生成自然的回答
关键理解：你不需要写任何循环逻辑！create_agent() 自动处理了「调用工具 → 拿结果 → 再推理」的循环。这是 LangChain Agent 的核心价值。

5. 面试常见追问

问题你的回答要点
Tool 的定义需要什么？ 函数名 + type hints（参数类型）+ docstring（描述工具用途）。LLM 通过这些来决定何时调用、传什么参数。
Agent 怎么知道调用哪个工具？ LLM 根据工具的描述（docstring）和参数类型自动判断。这就是 Function Calling 机制。
Agent 会无限循环调用工具吗？ 不会。LangChain 有最大迭代次数限制（默认），当 LLM 认为不需要工具时直接返回结果。

6. 动手练习

🛠 练习：扩展你的 Agent

  1. 添加第二个工具 get_time(city: str) -> str，返回城市当前时间（用模拟数据）
  2. 修改 system_prompt，让 Agent 优先尝试调用工具获取实时数据再回答
  3. 观察：当你问一个不需要工具的问题（如「你好，你是谁？」）时，Agent 的消息历史中是否还有 tool_call？

7. 关键 API 速记

API用途
create_agent(model, tools, system_prompt)创建 Agent 的最简方法
agent.invoke({"messages": [...]})运行 Agent，返回完整消息历史
init_chat_model(name, temperature)统一模型初始化（支持 OpenAI/Anthropic/Google 等）
agent.stream(...)流式运行 Agent（下一课讲）
