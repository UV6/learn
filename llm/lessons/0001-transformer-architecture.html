Lesson 0001: Transformer 架构

本课目标：理解 Transformer 为什么会出现，Self-Attention 的 Q/K/V 直觉是什么，Multi-Head Attention 为什么要多头，并能手画 Transformer 的完整架构图。

1. 为什么是 Transformer？

面试开场："Transformer 解决了 RNN/LSTM 的什么问题？"

RNN/LSTMTransformer
串行处理：第 3 个 token 必须等前 2 个处理完 并行处理：所有 token 同时计算 attention
长距离依赖衰减：第 1 个词和 第 100 个词的关系被 98 步"稀释" 常数路径：任意两个 token 通过 O(1) 的 attention 直接连接
梯度消失/爆炸（LSTM 缓解但未根除） Residual connection 让梯度顺畅流动
训练慢（时间步依赖） 训练快（矩阵乘法高度并行，GPU 友好）
面试标准答案："Transformer 用 Self-Attention 机制替换了 RNN 的循环结构，实现了三个突破：(1) 并行计算——所有位置同时计算 attention（2）长距离依赖——任意两个 token 之间的信息路径长度是 O(1)（3）可解释性——attention 权重直观展示 token 之间的关系。"

3. Self-Attention：本课核心

3.1 直觉

假设句子 "The cat sat on the mat because it was tired."

问题：「it」指什么？人类知道 it 指 cat，因为 cat 和 it 语义相关度高。

Self-Attention 做的事：为句子中每个词计算它与所有其他词的"相关分数"。

  • "it" 对 "cat" 的 attention 高 → 0.8
  • "it" 对 "mat" 的 attention 较低 → 0.15
  • "it" 对 "the" 的 attention 很低 → 0.05

3.2 从直觉到公式

面试时你不需要推导，但需要解释每个符号的含义：

Attention(Q, K, V) = softmax( Q·KT / √dk ) · V
术语直觉形状（以单头为例）
Q (Query) "我在找什么？" — 当前 token 的"需求"向量 [seq_len, d_k]
K (Key) "我是什么？" — 每个 token 的"标签"向量 [seq_len, d_k]
V (Value) "我包含什么信息？" — 每个 token 的实际内容 [seq_len, d_v]
Q·KT Query 和每个 Key 的"匹配分数"（点积越大越相关） [seq_len, seq_len] (注意力矩阵)
/√dk Scale 因子——防止点积值太大 → softmax 梯度消失 标量
softmax(...) 把分数变成概率分布（每行和为 1） [seq_len, seq_len]
·V 用注意力权重对 V 加权求和 → 输出 [seq_len, d_v]
面试点睛：用检索类比解释 Q/K/V——Q 是搜索词，K 是文档标签，V 是文档内容。搜索 → 匹配标签 → 拉取内容。Self-Attention 就是在句子内部做"检索"。

3.3 为什么除以 √dk ？

面试高频追问。答案：

当 dk 很大时（如 64），Q·KT 的值会很大。Softmax 对大输入值非常敏感——梯度趋近于 0（"饱和"）。除以 √dk 把数值范围拉回一个合理的区间，让 softmax 保持温和的梯度。

这是论文中少数的"工程技巧"——不是理论必然，而是实验结果。

4. Multi-Head Attention

单头的问题：一个 attention 只能捕捉一类关系。

多头 = 多组独立的 Q/K/V 投影 → 并行捕捉多种关系：

输入 X → 多组线性投影 → ┌─ WQ1,WK1,WV1 → Head1: Attention → output1 ├─ WQ2,WK2,WV2 → Head2: Attention → output2 ├─ ... └─ WQh,WKh,WVh → Head8: Attention → output8 → Concat [output1, ..., output8] → WO → 最终输出

面试理解：

  • Head 1 可能学会关注"语法关系"（动词→主语）
  • Head 2 可能学会关注"指代关系"（it→cat）
  • Head 3 可能学会关注"语义相似度"

每头处理 dk = d_model / h 维（如 512/8 = 64），总计算量和单头大维度相同，但表达能力更强。

5. 从 Transformer 到 GPT：为什么只要 Decoder？

GPT 是 decoder-only 架构。它扔掉了 Encoder 和 Cross-Attention，原因：

  1. 自回归生成 = Decoder 的天生能力：Masked Self-Attention 确保每个 token 只看左边，完美匹配"一个接一个"的生成方式
  2. Encoder 的双向注意力不匹配生成：Encoder 能看到所有 token（包括"未来"），生成时不可用
  3. Scaling Law 的发现：OpenAI 发现只管堆 decoder 层数和数据，效果比复杂的 encoder-decoder 更好
面试金句："GPT 选择 decoder-only 不是理论必然，而是工程实践的收敛——自回归 + 大规模数据 + 简单的架构 = 涌现能力。LLaMA、Mistral、Falcon 都遵循了这一范式。"

6. Feed Forward Network (FFN)：Transformer 的"知识库"

Attention 负责"检索"（token 之间怎么交互），FFN 负责"存储"（每个 token 内部的信息处理）。

FFN(x) = GELU(x·W1 + b1)·W2 + b2

两个关键事实（面试必知）：

  • FFN 占 Transformer 总参数的 2/3 — 因为中间维度膨胀到 4×d_model
  • FFN 存储了模型的知识 — 研究表明 FFN 的权重编码了事实性知识（"巴黎是法国首都"），而 Attention 编码了 token 之间的关系

7. 面试高频追问

问题你的回答要点
Attention 的复杂度？ O(n²·d)，n 是序列长度，d 是维度。n² 是瓶颈——当 n=128K 时不可接受。所以有 Flash Attention、Sparse Attention、Linear Attention 等优化。
LayerNorm vs BatchNorm？ BatchNorm 在 batch 维度做归一化（受 batch size 影响），LayerNorm 在 feature 维度做（不受 batch size 影响）。NLP 中序列长度不一，LayerNorm 更稳定。Transformer 选 LayerNorm 是必然的。
Post-Norm vs Pre-Norm？ 原始论文用 Post-Norm (x + LayerNorm(Sublayer(x)))，但现代模型（LLaMA 等）都用 Pre-Norm (x + Sublayer(LayerNorm(x)))。Pre-Norm 训练更稳定，梯度不容易爆炸。
Positional Encoding 为什么需要？ Self-Attention 是置换不变的——"我爱你"和"你爱我"的 attention 矩阵完全相同。Positional Encoding 给每个位置注入一个标识向量，让模型知道 token 的顺序。
Encoder-Decoder Attention 是什么？ Decoder 的 Cross-Attention 层。它的 Q 来自 Decoder（我要生成什么），K/V 来自 Encoder（源语言提供了什么信息）。翻译模型用这个，GPT 不需要。
为什么现在都用 decoder-only？ 3 个原因：(1) Scaling Law——简单架构 + 更多数据 > 复杂架构 (2) 自回归统一了预训练和下游任务——next-token prediction 就是一切 (3) 工程简洁——一种架构、一种训练目标、无微调切换。

8. 动手练习

🛠 练习：内化 Attention

  1. 手推一个例子：取 3 个 token、d_k=2，手动算一遍 Q·KT、scale、softmax、× V。感受一下数字是怎么流动的。
  2. 手画架构图：不参考课件，默画 Transformer 的完整架构图。标注每个箭头旁边张量的形状。
  3. 读代码：去 Andrej Karpathy 的 nanoGPT，看 model.py 里的 CausalSelfAttention 类——看看 Q/K/V 的代码如何对应公式。
  4. 面试自测：用 2 分钟口头解释 "Self-Attention 是什么"，录下来回听。目标是清晰、不结巴、有公式也有直觉。
