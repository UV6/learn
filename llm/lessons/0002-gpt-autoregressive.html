Lesson 0002: GPT 自回归生成

2026-06-30 · 预计时间 35 分钟 · 难度：中级偏高

本课目标：理解 GPT 的自回归生成机制——为什么是 next-token prediction、masked attention 怎么工作、temperature/top-p/top-k 控制什么、以及训练和推理阶段的区别。

1. GPT 的核心思想：一切皆是 next-token prediction

面试最常问："GPT 的训练目标是什么？"

答案：自回归语言建模 (Autoregressive Language Modeling)——给定前面的 token，预测下一个 token。

P(x1, x2, ..., xn) = ∏ P(xi | x1, ..., xi-1)

训练时，给模型前半句，让它预测后半句。这个简单的目标在足够大的数据上训练后，产生了惊人的能力。

关键洞察：GPT 把所有 NLP 任务都统一为 next-token prediction。翻译？"Translate to French: Hello → " → 预测 "Bonjour"。总结？"Summarize: [长文] → TL;DR:" → 预测摘要。代码生成？"def fib(n):" → 预测函数体。这就是 in-context learning 的本质——不需要改模型，只需要改 prompt。

2. Masked Self-Attention：只看左边

GPT 不能偷看未来。Masked Attention 通过一个上三角 mask 矩阵实现：

句子："I love AI" Tokens: [I, love, AI] 注意力矩阵 (Q·KT)： + Mask (-∞ 上三角)： I love AI I love AI I [0.8 0.5 0.3] I [0.8 -∞ -∞ ] love [0.5 0.9 0.7] → love [0.5 0.9 -∞ ] AI [0.3 0.7 0.9] AI [0.3 0.7 0.9] 经过 softmax(mask)： 含义： I [1.0 0 0 ] I 只能看到自己 love [0.3 0.7 0 ] love 能看到 I 和自己 AI [0.2 0.3 0.5] AI 能看到所有前面的词

这就是 Chat GPT "一个词一个词往外蹦"的原因——每生成一个 token，它就追加到序列末尾，成为下一个预测的输入。

4. 采样策略：控制生成的"创造性"

GPT 每一步输出的是概率分布（vocab_size 维的向量）。怎么从分布中选一个 token？

策略做法效果
Greedy 每次选概率最高的 token 确定性的，但容易重复、无聊
Temperature softmax(z / T)，T>1 拉平分布，T<1 锐化分布 T=0.7 温和创造；T=1.5 放飞自我
Top-K 只从概率最高的 K 个 token 中采样 K=50：排除低概率的噪音 token
Top-P (Nucleus) 从累积概率 ≥ P 的最小集合中采样 比 Top-K 更灵活——动态调整候选集大小
生产环境最佳实践：Temperature=0（greedy）用于事实性任务（代码、数学），Temperature=0.7 + Top-P=0.9 用于创意写作。Top-K 和 Top-P 不要同时用——通常 Top-P 更优。

5. 为什么 GPT 能"理解"指令？从 GPT-1 到 GPT-4 的演进

模型关键创新解决什么问题
GPT-1 (2018)预训练 + 有监督微调证明 Transformer decoder 做语言建模有效
GPT-2 (2019)Zero-shot transfer + 更大的规模"语言模型就是多任务学习器"——不需要微调
GPT-3 (2020)In-context learning (175B)few-shot prompt 替代微调，涌现能力出现
GPT-3.5/InstructGPTRLHF (人类反馈强化学习)对齐——让模型遵循意图、拒绝有害请求
GPT-4 (2023)多模态 + 更大的规模图像理解、更复杂的推理
面试框架：GPT 的演进是三个阶段——(1) 预训练阶段：学习世界知识，next-token prediction (2) 对齐阶段：RLHF 让人工偏好注入模型 (3) 推理阶段：通过 prompt engineering 和 tool use 扩展能力。面试官想听的是你理解这三个阶段的边界。

6. 面试高频追问

问题你的回答要点
Teacher Forcing 是什么？ 训练时把正确答案（而非上一步的输出）作为下一步的输入。即使模型预测错了也不影响后续——因为 ground truth 被"强制"喂入。这让训练并行，但造成了训练-推理的分布偏差（exposure bias）。
GPT 为什么不 bidirectional？ 生成任务是因果性的——你不能用未来的词来预测当前词（信息泄露）。BERT 做了双向 attention，但它不做生成。如果用 BERT 生成，每个 token 的预测是独立的，没有连贯性。
自回归 vs 非自回归？ 自回归：逐 token 生成，慢但连贯。非自回归：一次输出全部 token，快但质量差。目前所有主流 LLM 都是自回归的，非自回归是研究热点但未实用。
为什么 GPT-3 的 few-shot 有效？ 大规模预训练让模型学会了"模式匹配"——给定 few-shot example 后，模型在 context 中找到了最匹配的 pattern，直接复现。这不是真的"学习"，而是更高效的 pattern completion。

7. 动手练习

🛠 练习

  1. 手算 softmax + temperature：给定 logits=[2.0, 1.0, 0.1]，分别算 T=1.0 和 T=0.5 时的概率分布，体会温度的影响
  2. 运行 nanoGPT：clone Karpathy 的 nanoGPT，跑一次推理，观察 next-token prediction 的每一步输出
  3. 对比采样：同一个 prompt，分别用 greedy、T=0.7、T=1.5 生成三次，观察何时开始分叉

8. 关键术语

Autoregressive生成方式：P(tₙ | t₁, ..., tₙ₋₁)，一步一个
Teacher Forcing训练时用 ground truth 作为输入，加速训练
Masked Attention上三角 -∞ mask，防止看到未来 token
Temperature控制 softmax 的"锐度"，调创造性
Top-P Sampling从累积概率 ≥ P 的集合中采样，动态截断
In-Context Learning通过 prompt 中的示例"教会"模型，无需微调
RLHF用人类偏好训练 reward model，再优化 policy
EOS Token序列结束标记，告诉模型"停"
