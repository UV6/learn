Lesson 0003: LLaMA 的工程改进

2026-06-30 · 预计时间 40 分钟 · 难度：中高级

本课目标：理解 LLaMA/LLaMA 2 相比原始 Transformer 的四个关键改进——RoPE、RMSNorm、SwiGLU、GQA——以及它们解决了什么问题。

1. 原始 Transformer → 现代 LLM 的进化

LLaMA (Meta, 2023) 是当前最流行的开源 LLM 架构。它的设计哲学是：保持核心不变，优化每一个组件。

组件原始 TransformerLLaMA为什么改？
位置编码 Sinusoidal / Learned RoPE 更好的外推（训练 2K，推理 32K）
归一化 LayerNorm (Post-Norm) RMSNorm (Pre-Norm) 更快、更稳定
激活函数 ReLU / GELU SwiGLU 更好的训练效果
Attention Multi-Head Attention GQA (Grouped Query Attention) 推理时省 KV Cache

2. RoPE：旋转位置编码

问题

Sinusoidal Positional Encoding 是"加到 embedding 上"的绝对位置编码。它不能很好地处理相对位置——"cat"和"sit"之间的距离信息在 attention 计算中丢失了。

Learned Positional Encoding 则无法外推——如果训练时最大长度 2048，推理时输入 5000 token 就会出错。

RoPE 的直觉

RoPE (Rotary Position Embedding) 的核心思想：通过旋转向量来编码位置，让两个向量的点积自然地包含相对位置信息。

f(Q, m) = Q · Rm,   f(K, n) = K · Rn
f(Q, m)·f(K, n)T = Q·Rm-n·KT

Rm 是一个旋转矩阵，把向量按维度对的频率旋转 m 个角度。结果是：Q 和 K 的点积只依赖它们的相对位置 (m-n)。

面试精炼："RoPE 通过旋转变换把绝对位置转换成相对位置信息。它有两个优势：(1) 相对位置自然地出现在 attention 计算中 (2) 支持长度外推——训练 2K 推理 8K 甚至 32K，通过插值频率或在长数据上继续训练即可。"

RoPE 的实际表现

  • 频率随维度指数衰减——低维度捕捉长距离依赖，高维度捕捉局部依赖
  • 外推能力来自"插值"——把位置 ID 线性缩小，让模型"以为"自己仍在训练长度内
  • 已被几乎所有开源模型采用：LLaMA, Mistral, Qwen, DeepSeek, InternLM

3. RMSNorm：轻量级归一化

LayerNorm 做了什么

LayerNorm(x) = γ · (x - μ) / σ + β

μ 是均值，σ 是标准差。减去均值 + 缩放——计算量大。

RMSNorm 的简化

RMSNorm(x) = γ · x / RMS(x),   RMS(x) = √(mean(x²))

只除以 RMS（均方根），不减均值。实验证明：

  • 效果和 LayerNorm 几乎一样
  • 计算量减少约 10-15%
  • 梯度更稳定（没有均值减法的漂移）

Pre-Norm

原始 Transformer：x + Sublayer(LayerNorm(x)) (Post-Norm, 残差外面做 norm)

LLaMA：x + Sublayer(RMSNorm(x)) (Pre-Norm, 残差里面做 norm)

Pre-Norm 让训练更稳定——梯度不经过 LayerNorm 的缩放，直接从残差通路传播。现代模型全用 Pre-Norm。

4. SwiGLU：更好的激活函数

从 ReLU 到 SwiGLU

激活函数公式问题
ReLUmax(0, x)负区间完全死掉（梯度=0）
GELUx·Φ(x) 近似平滑但没有门控
SwiGLU(x·W1 · Swish(x·W2))·W3门控 + 非线性 = 更好
直觉：SwiGLU 引入了一个"门控"机制——一个线性变换的结果经过 Swish 后作为"门"去控制另一个线性变换的结果。类似于 LSTM 的门控思想，但更简单高效。PaLM、LLaMA、Gemma 的实验都证明了它优于 GELU。

值得一提的是：SwiGLU 的中间维度通常设为 8/3 × d_model（而非原始 FFN 的 4×），因为有三个权重矩阵而非两个，需要缩小以保持参数量相近。

5. GQA：Grouped Query Attention

动机：KV Cache 是推理瓶颈

Multi-Head Attention 每层存 h 组 KV 向量。对于 LLaMA 70B（80 层，h=64, d_model=8192），单个 token 的 KV Cache 约 2.5GB。长上下文推理时这就是内存瓶颈。

三种 Attention 对比

MHA (Multi-Head)MQA (Multi-Query)GQA (Grouped Query)
Q 头数hhh
KV 头数h1g (1 < g < h)
KV Cache大（h 份 KV）小（1 份 KV）中等（g 份 KV）
质量最好有损接近 MHA
MHA: Q₁Q₂Q₃Q₄ → K₁K₂K₃K₄ → 每头独立 KV (4 份) MQA: Q₁Q₂Q₃Q₄ → K (共享) → 所有头共享 1 份 KV GQA: Q₁Q₂│Q₃Q₄ → K₁│K₂ → 每 2 头共享 1 份 KV (2 份)

LLaMA 2 70B 用 GQA(g=8, h=64)，KV Cache 缩小 8 倍，推理质量无明显下降。

面试金句："GQA 是 MHA 和 MQA 的折中——保留了多头 Q 的表达能力，同时减少 KV Cache。这展示了现代 LLM 设计的核心 tradeoff：推理效率 vs 模型质量。LLaMA 2 论文证明了 g=8 几乎无损。"

6. 四个改进的协同效应

LLaMA 的设计不是随机堆砌改进——它们之间形成了协同：

  • RoPE + GQA：RoPE 提供长上下文的位置编码 → GQA 减少长上下文的 KV Cache 开销 → 组合后长上下文推理变得可行
  • RMSNorm(Pre-Norm) + SwiGLU：Pre-Norm 让深层梯度稳定 → SwiGLU 在稳定梯度下更好地学习 → 可以堆更深（70B 模型)的层

7. 面试高频追问

问题你的回答要点
RoPE vs ALiBi？ 两种都支持长度外推。ALiBi 是加 bias 到 attention score（简单粗暴），RoPE 是旋转位置编码（更优雅）。RoPE 在长上下文微调后效果更好，ALiBi 零样本外推更强。当前趋势是 RoPE 胜出。
为什么不用 BatchNorm？ BatchNorm 依赖 batch 统计量，推理时用训练的平均值。NLP 序列长度不一，batch 统计量不稳定。LayerNorm/RMSNorm 在 feature 维度归一化，不受序列长度和 batch size 影响。
SwiGLU 比 GELU 好多少？ 同等参数量下，SwiGLU 的 perplexity 下降约 2-5%。关键不是绝对数值，而是它引入了 gating——让模型可以"选择性"地传递信息，类似 LSTM 的门控。
LLaMA 和 GPT 的架构差异？ 本质都是 decoder-only transformer。GPT-3 用 Learned PE + LayerNorm + GELU + MHA；LLaMA 用 RoPE + RMSNorm + SwiGLU + GQA。LLaMA 的改进都是工程优化，不改变自回归生成的核心逻辑。
