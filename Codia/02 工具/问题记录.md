## 2026-06-23

### 工具结果在 TUI 中显示完整文件内容，改用摘要

**问：** 工具调用后 TUI 会显示完整的工具返回内容（如 Read 显示整个文件），能否改为 "📖 读取了 xxx 文件" 这样的紧凑摘要？

**答：** 实现方案涉及 4 个文件：

1. **`src/provider/types.ts`** — Message.toolResults 扩展 `name` 和 `inputPreview` 字段
2. **`src/agent/loop.ts`** — 打包工具结果时从 `result.toolCalls` 构建 callId→ToolCall 查找表，用 `buildInputPreview()` 函数提取展示参数（filePath/pattern/command）
3. **`src/tui/chat-view.tsx`** — 新增 TOOL_ICONS 映射和 `renderToolSummary()` 函数，检测 `msg.toolResults` 存在时渲染摘要行替代原始 content
4. **`src/tui/app.tsx`** — 新增 `expandedTools` state + Ctrl+E 快捷键切换展开/折叠

各工具摘要格式：
| 工具 | 摘要 |
|------|------|
| read_file | 📖 读取了 path (N 行) |
| glob | 🔍 搜索 "pattern" 匹配 N 个文件 |
| grep | 🔎 搜索 "pattern" 找到 N 处匹配 |
| write_file | ✏️ 写入了 path (N B) |
| edit_file | ✂️ 编辑了 path |
| run_command | 💻 cmd (exit: N, X.Xs) |
| 未知/MCP | 🔧 name 完成 |

LLM 侧不受影响——完整 content 仍然通过消息历史发送，只有 TUI 显示层做了折叠。
