### MCP 配置层级

**问：** 当前项目怎么配置 MCP 的？mcp 有几个配置的地方？

**答：** Codia 的 MCP 配置有 **2 个层级**，在 `src/mcp/config.ts` 中由 `loadMcpConfig()` 加载合并：

| 层级 | 路径 | 作用 |
|------|------|------|
| 用户级 | `~/.Codia/Codia.yml` | 全局默认 |
| 项目级 | `<project>/.codia/config.yml` | 项目专属，覆盖用户级同名 Server |

合并流程：读两个 YAML 的 `mcp_servers` 段 → 项目级覆盖用户级 → `${VAR}` 环境变量展开 → 校验（不合法则跳过）

支持的两种类型：
- **stdio**: command + args + env → `StdioTransport` 子进程通信
- **http**: url + headers → `HttpTransport`

运行时入口：`ChatService.init() → loadMcpConfig() → ConnectionManager.connectAll()`，对每个 Server 用 `McpClient.connect()` 建立连接、`fetchTools()` 获取工具列表，再用 `createMcpToolAdapter` 注册到 `ToolRegistry`（以 `mcp__<server>__` 前缀命名）。

### 配置 Context7 MCP

**问：** 生成一个项目级的 config.yml，并在里面根据 codia 的配置规则加上 context 7 的配置。

**答：** 创建 `/Users/liuwei/Code/Codia/.codia/config.yml`，内容：

```yaml
mcp_servers:
  context7:
    type: stdio
    command: npx
    args:
      - "-y"
      - "@upstash/context7-mcp@latest"
```
