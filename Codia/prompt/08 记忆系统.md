# 我的初步想法
这一步的目标是：在新会话启动时自动恢复项目知识和用户偏好，让 Agent 从「每次失忆」变成「越用越懂你」。靠项目指令文件、会话存档和自动记忆三套机制，实现工作记忆和长期记忆的分层管理，中断之后也能平滑接着用。

技术要求：

- 项目指令文件支持多层优先级（项目级高于用户级），高优先级排前面让模型优先遵循；支持 @include 引用其他文件，但要限嵌套深度、用 visited 集合防环路、拦截跳出项目目录的路径
- 会话用 JSONL 追加写：追加快、崩溃只丢最后一行、恢复时坏行可跳过；不单独维护 meta 文件，需要 ID/标题/消息数时直接扫 JSONL 算出来，少一份要保持同步的状态
- 会话恢复要处理几类异常：坏行跳过、工具调用没配上结果时截断、token 超限先压一次、隔太久插一条时间跨度提醒；30 天以上的过期会话定期清理
- 自动笔记分四类（用户偏好、纠正反馈、项目知识、参考资料），每轮 Agent Loop 自然停下后（模型最终回复无工具调用时）异步调 LLM 更新，去重交给 LLM 判断
- 用户级和项目级笔记分开存，记忆索引在处理请求前就注入上下文，相当于 Agent 已经「读过」；索引文件控制在 200 行 / 25KB 以内（约 2-3K tokens）

这一步先不做向量数据库、RAG 检索、团队记忆同步，留给后续章节或专门项目。

记忆文件格式：

- 项目指令文件：三层加载，项目根、项目根下 .mewcode/、用户目录 ~/.mewcode/ 各一份手写 Markdown，记技术栈、代码规范、注意事项，按优先级拼接
- 会话存档：放在项目的 sessions 目录，每个会话一个 JSONL 文件；会话 ID 用 `YYYYMMDD-HHMMSS-xxxx` 防同秒撞车
- 自动笔记：放在 memory 目录，每条笔记一个带 frontmatter 的 Markdown，另有一份索引文件
