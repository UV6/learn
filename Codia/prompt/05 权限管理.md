# 我的初步想法
这一步的目标是：给 Codia 装上一套五层防御的权限系统。黑名单硬拦截危险命令，沙箱限制文件操作范围，规则引擎做细粒度控制，权限模式覆盖从严格到放行的信任等级，人在回路作为最后确认。权限被拒绝时不终止 Agent Loop，让模型有机会调整策略。

技术要求：

- 危险操作黑名单：用正则在执行前拦掉已知高危命令，这层不可被配置放开
- 路径沙箱：限制文件读写只能落在项目目录内，要先解析符号链接再做前缀判断防逃逸
- 可配置规则：按「工具加参数或路径模式」声明放行还是拦截，支持精确和 glob 匹配
- 多档权限模式：让用户整体切换严格、默认、放行等档位，覆盖在具体规则之上
- 人在回路：规则没明确命中时把决定权交回用户，支持本次、本会话、永久三种放行
- 规则优先级：会话级临时规则盖过项目级，项目级盖过用户全局默认，逐层判断

这一步先不做网络请求限制、资源配额、审计日志，留给后续章节。

规则配置格式：

- 规则写成「工具名(模式)」，例如 Bash(git *) 表示放行所有 git 命令
- 每条规则的结果只有 allow 或 deny 两种
- 规则按用户级、项目级、本地级三层 YAML 文件分别存放，越靠近项目优先级越高
