### 思考完成后自动折叠 thinking 内容

**问：** 思考完成后，当输出回答的时候，思考的内容仍存在并占用了一定的篇幅，想要思考完成后思考的内容直接折叠了。

**答：** 在 `src/tui/app.tsx` 的 `handleAISubmit` 中，当收到第一个 `text` chunk 时，如果之前有 thinking 内容积累（`fullThinking` 非空），自动调用 `setThinkingCollapsed(true)` 折叠。用 `thinkingAutoCollapsed` 局部变量避免重复调用 setState。

关键改动（`app.tsx:115-118`）：
```typescript
case "text":
  if (fullThinking && !thinkingAutoCollapsed) {
    setThinkingCollapsed(true);
    thinkingAutoCollapsed = true;
  }
  // ...
```

折叠后 ThinkingBox 显示一行 `<Thinking... (Ctrl+T 展开)>`，用户可按 Ctrl+T 重新展开查看。

### Ctrl+T/Ctrl+E 快捷键被 TextInput 吞掉字符

**问：** 按 Ctrl+T 只在输入框输入了 "t"，并没有识别到这个快捷键。

**根因：** **Ink 不支持事件 stopPropagation。** 所有组件的 `useInput` 都会独立触发，互不屏蔽。当按 Ctrl+T 时，两个 handler **同时**运行：

| 顺序 | 组件 | 触发行为 |
|------|------|----------|
| 1 | InputBox 的 `useInput` | 调用 `onToggleThinking`，切换折叠 ✓ |
| 2 | ink-text-input 内部的 `useInput` | 把 "t" 当作普通字符写入 state ✗ |

`ink-text-input` 内部不检查 `key.ctrl`，Ctrl 修饰键对它透明——它只看普通字符 "t" 就执行 `onChange`。所以即使 InputBox 正确处理了快捷键，输入框里仍然会出现 "t"。

**修复：** 两步配合解决。

1. 在 InputBox 的 `useInput` 中拦截 Ctrl+T/Ctrl+E（该 handler 先于 TextInput 内部的 handler 触发），设置 `suppressChangeRef = true` 并执行切换
2. 用 `handleChange` 包装 TextInput 的 `onChange`：当 `suppressChangeRef` 为 true 时，跳过更新并重置 ref——对应字符被无声吞掉

关键代码：
```typescript
const suppressChangeRef = useRef(false);

const handleChange = (newValue: string) => {
  if (suppressChangeRef.current) {
    suppressChangeRef.current = false;
    return; // 吞掉，不写入值
  }
  setValue(newValue);
};

useInput((input, key) => {
  if (key.ctrl && input === "t") {
    suppressChangeRef.current = true;
    onToggleThinking?.();
  }
  // ...
});

// TextInput 改用包装后的 handleChange
<TextInput value={value} onChange={handleChange} ... />
```

**总结：** Ink 不支持事件冒泡/阻断，任何快捷键方案都必须是：拦截 → ref 标记 → 吞掉后续的普通字符写入。纯靠 `useInput` 顺序不够。
