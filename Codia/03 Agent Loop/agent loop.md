**循环终止条件**
- 模型主动说做完了：Codia 返回的 stop_reason 是 end_turn，且响应里没有任何tool_use，就表示模型认为已经完成
- 迭代达到上限：设置一个最大循环次数，比如五十次，当达到最大循环次数时，告诉用户已经执行了 50 步还没完成，已自动停止。
- 用户取消：用户按 esc 终止循环，需要支持取消信号传播，用户在 UI 层触发取消，信号传递到Agent Loop，并在下一次循环前检测是否取消，取消了就直接退出。
- 异常状态检测：如果模型调用的工具出错了，比如说不存在，或者被禁用了，就返回一个错误结果让其自己调整。如果连续多次都请求不存在的工具，说明模型已经迷失了，可以提前终止。比如说设置连续三次找不到请求的工具，就提前终止。

⭐️**工具执行的分批处理**
如果一次调用多个工具，可以根据是否并行安全来并行执行，比如说 read。如果涉及到写操作或破坏性的操作就串行执行。同时设置最大并行上限来防止无限并发。

**System Prompt 与 环境信息**
Loop 中每次循环都需要带上 System 提示词，以及当前项目目录，环境、操作系统、时间等环境信息，不知道目录就不知道在哪里执行命令，不知道操作系统就可能在 macos 上执行 windows 命令，这些可以放到 prompt cache 里![[Pasted image 20260610141506.png|463]]
