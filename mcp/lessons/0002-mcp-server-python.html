本课目标：用 MCP Python SDK 从零实现一个 MCP Server，提供 Tool + Resource，并配置到 Claude Code 中实际使用。

1. 项目初始化

mkdir my-first-mcp && cd my-first-mcp
python -m venv .venv
source .venv/bin/activate
pip install mcp

SDK 的核心类是：

  • mcp.server.Server — MCP Server 主类
  • mcp.types.Tool — 工具定义（name, description, inputSchema）
  • mcp.types.Resource — 资源定义（uri, name, description, mimeType）
  • mcp.server.stdio.stdio_server() — 通过 stdio 传输运行

2. 最简 MCP Server：Hello World

创建 server.py：

#!/usr/bin/env python3
"""第一个 MCP Server — 提供天气查询和笔记读取"""

import json
from mcp.server import Server, NotificationOptions
from mcp.server.models import InitializationCapabilities
from mcp.server.stdio import stdio_server
from mcp.types import (
    Tool,
    TextContent,
    Resource,
    ResourceTemplate,
)

# Step 1: 创建 Server 实例
server = Server("weather-server")


# Step 2: 定义能力
@server.list_tools()
async def handle_list_tools() -> list[Tool]:
    """告诉 Client 我有什么工具"""
    return [
        Tool(
            name="get_weather",
            description="查询指定城市的当前天气",
            inputSchema={
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称"
                    }
                },
                "required": ["city"]
            }
        ),
        Tool(
            name="get_time",
            description="查询指定城市的当前时间",
            inputSchema={
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称"
                    }
                },
                "required": ["city"]
            }
        ),
    ]


# Step 3: 实现工具调用
@server.call_tool()
async def handle_call_tool(
    name: str, arguments: dict
) -> list[TextContent]:
    """处理工具调用请求"""

    if name == "get_weather":
        city = arguments["city"]
        # 实际项目调天气 API
        weather_data = {
            "beijing": "sunny, 25°C",
            "shanghai": "cloudy, 28°C",
        }
        result = weather_data.get(
            city.lower(), f"No data for {city}"
        )
        return [TextContent(type="text", text=result)]

    if name == "get_time":
        from datetime import datetime
        city = arguments["city"]
        return [TextContent(
            type="text",
            text=f"Current time: {datetime.now().isoformat()}"
        )]

    raise ValueError(f"Unknown tool: {name}")


# Step 4: 提供 Resource（可被 Client 读取的数据）
@server.list_resources()
async def handle_list_resources() -> list[Resource]:
    """告诉 Client 我有什么数据资源"""
    return [
        Resource(
            uri="weather://cities",
            name="Supported Cities",
            description="支持查询的城市列表",
            mimeType="application/json",
        ),
    ]


@server.read_resource()
async def handle_read_resource(uri: str) -> str:
    """读取资源内容"""
    if uri == "weather://cities":
        return json.dumps(
            ["Beijing", "Shanghai", "Tokyo", "London"],
            ensure_ascii=False,
        )
    raise ValueError(f"Unknown resource: {uri}")


# Step 5: 启动 Server
async def main():
    async with stdio_server() as (read_stream, write_stream):
        await server.run(
            read_stream,
            write_stream,
            InitializationCapabilities(
                sampling=None,  # 本 Server 不需要调用 LLM
                experimental=None,
            ),
        )

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())

3. 配置到 Claude Code

在项目根目录创建 .mcp.json：

{
  "mcpServers": {
    "weather": {
      "command": "python",
      "args": ["server.py"],
      "cwd": "${workspaceFolder}"
    }
  }
}

重启 Claude Code 后，Agent 自动发现 get_weather 和 get_time 两个新工具。

你已经在用 MCP：你的 Claude Code 里配置了 codegraph、playwright、context7 等 MCP 工具——它们就是这样运行的。你刚写的 server.py 和它们在一个水平上。

4. 代码走读：四个关键钩子

装饰器触发时机返回值
@server.list_tools()Client 首次连接时Tool 列表（name + description + inputSchema）
@server.call_tool()LLM 决定调用某个工具时Tool 执行结果（TextContent 或 ImageContent）
@server.list_resources()Client 请求资源列表时Resource 列表（uri + name + mimeType）
@server.read_resource()Client 读取某个资源时资源内容（字符串或 bytes）

和 LangChain 的 @tool 装饰器对比：

  • LangChain @tool：装饰一个 Python 函数，自动提取 type hints 生成 schema
  • MCP @server.call_tool：手动处理 name 和 arguments，需要显式 dispatch
  • 关键差异：MCP 的工具定义和工具实现是分离的——list_tools 返回 schema，call_tool 返回结果。LangChain 把它们合在一起。

5. 进阶：带认证的 HTTP MCP Server

stdio 适合本地开发，HTTP 适合共享服务：

# 如果用 FastMCP (更高层封装)
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("Weather Server")

@mcp.tool()
async def get_weather(city: str) -> str:
    """查询城市天气"""
    return f"Sunny, 25°C in {city}"

@mcp.resource("weather://cities")
async def list_cities() -> str:
    """支持的城市列表"""
    return json.dumps(["Beijing", "Shanghai"])

# 一行启动 HTTP 服务
if __name__ == "__main__":
    mcp.run(transport="sse", port=8000)

FastMCP 封装了 tools/list 和 call_tool 的样板代码——更像 LangChain 的 @tool 风格。

6. 面试高频追问

问题你的回答要点
"MCP SDK 和 FastMCP 的区别？" SDK 是底层 API（手动 dispatch + 显式 schema 定义），FastMCP 是高层封装（@tool 自动生成 schema）。SDK 给框架开发者用，FastMCP 给应用开发者用。类比：SQLAlchemy Core vs ORM。
"MCP 工具怎么处理错误？" 返回一个包含错误信息的 TextContent（不要抛异常）。MCP 协议的 error 字段用于协议层错误（如 method not found），业务错误应该在 result 中返回。
"MCP 工具可以异步吗？" 可以，所有 handler 都是 async 的。Tool 调用本身就是 async——SDK 默认支持。
