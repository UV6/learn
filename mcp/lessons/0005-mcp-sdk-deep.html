1. Python SDK vs TypeScript SDK

MCP 官方维护两个 SDK——Python（更简洁）和 TypeScript（更完整）。你需要知道核心差异：

Python SDKTypeScript SDK
底层库mcp (pip)@modelcontextprotocol/sdk (npm)
高层封装FastMCPMcpServer (BaseServer 子类)
工具定义装饰器 + 函数签名自动推断 schemaserver.tool(name, description, schema, handler)
类型系统Pydantic modelsZod schemas
Resource 管理@mcp.resource(uri)server.resource(name, uri, handler)
Streamable HTTP实验性支持正式支持
生态成熟度略领先（AI/ML 生态用 Python 更多）功能更全（Web/JS 生态）
选择建议：如果你的 MCP Server 需要调用 Python ML 库（如 langchain, huggingface）→ Python SDK。如果你的 MCP Server 是 Node.js 服务的一部分 → TypeScript SDK。纯工具类（文件、API 封装）→ 两种都可以，选你更熟悉的那门语言。

2. Sampling：让 MCP Server 反过来调用 LLM

MCP 协议支持双向能力——不仅 Client 可以调 Server 的工具，Server 也可以请求 Client 帮忙调用 LLM（Sampling）：

# Server 端代码：请求 Client 帮忙生成文本
from mcp.types import CreateMessageRequest, SamplingMessage

@server.call_tool()
async def handle_summarize(name, arguments):
    """这个工具本身需要 LLM 来生成摘要"""

    # 请求 Client 帮我调用 LLM
    result = await server.request_sampling(
        CreateMessageRequest(
            messages=[
                SamplingMessage(
                    role="user",
                    content=TextContent(
                        type="text",
                        text=f"Summarize: {arguments['text']}"
                    ),
                ),
            ],
            maxTokens=200,
        )
    )

    # result.content 是 LLM 生成的摘要
    return [TextContent(type="text", text=result.content)]

使用场景：

  • Agent 链：一个 MCP Server 调用另一个 MCP Server 的 LLM 能力
  • 内容生成：Server 需要 LLM 完成子任务（摘要、翻译、格式转换）
  • 动态 Prompt：Server 根据运行时条件构建 prompt，委托 Client 执行
注意：Sampling 需要 Client 在 initialize 时声明 capabilities.sampling。不是所有 MCP Client 都支持 Sampling——Claude Desktop 支持，但部分第三方 Client 可能不支持。使用前需要做能力协商。

3. Notifications：Server 主动推送

MCP 的 Notification 机制让 Server 可以主动告知 Client 状态变化：

# Server 端：工具列表变更时通知 Client
await server.request_context.session.send_notification(
    "notifications/tools/list_changed"
)

# Server 端：资源更新时通知 Client
await server.request_context.session.send_notification(
    "notifications/resources/updated",
    {"uri": "orders://all"}  # 告知 Client 哪个资源变了
)

Client 端收到通知后通常会重新调用 tools/list 或 resources/read，保持 UI 同步。

4.2 分页返回

@server.list_resources()
async def list_resources(cursor: str | None = None):
    all_resources = get_all_files()  # 可能有 10000 个
    page_size = 100

    if cursor:
        start = int(cursor)
    else:
        start = 0

    page = all_resources[start : start + page_size]
    next_cursor = str(start + page_size) if len(page) == page_size else None

    return ListResourcesResult(
        resources=page,
        nextCursor=next_cursor,
    )
